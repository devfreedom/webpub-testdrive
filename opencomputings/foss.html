What's a computer?

We briefly mentioned some things like software, hardware, and source code. Now it's time to elaborate a little bit more.

First, what's a computer? Computer is a machine that can be programmed to compute logic or arithmetic automatically. If a computer does such computing electronically, not mechanically like cogs and wheels, then it's a digital computer. A modern digital computer is made of softwares and hardwares that can process and store information, through inputs and outputs.

Software

Software is a set of instruction to perform a certain task. The task here can by anything, from simply showing a text "hello!" to running a complicated scientific simulation. Any computer program we use on your desktop and laptop is software. Any mobile application we use every day is software. The fundamental software that runs all other softwares is also software and we call it "operating system", notably Windows, Android, Linux, and iOS.

Now back to "a set of instruction". For instance, an alarm clock app would need instructions like "check the current time, and if it's 3:30pm then play a ding-dong sound". To make your phone perform this, we humans have to write this instruction and put it inside your phone. This "human-readable version" of instructions written in a structured programming language is called "source code".

Every software is made of many lines of source code, again, instructions. Your computing device takes this given source code, and transforms it into "machine-readable" instructions so that they, not us, can understand and follow it as-is. This is how software works.

Software doesn't have a physical form. It exists virtually and runs digitally inside your device, so it's flexible, hence "soft"ware. We can easily change and control it because it's just a set of orders that we feed to the computer.

Hardware

Hardware is a set of physical parts of your computer. Your smartphone is a hardware. It is made of smaller hardwares like camera sensor, touchscreen display, battery, and central processing unit. Each hardware is ready-made to perform a specific task. Camera sensor is built to accept the light coming through a lens, nothing else. Central processing unit is there to do the major digital calculation and computation and that's it. We can't change its purpose and function after it's made. This is why we call it "hard"ware.

Hardwares can't do anything by itself because it is not linked to the software we had written. So hardwares also need a software that runs the hardware itself and interact with other softwares. This middleman software is called "firmware" because it's neither fully hard nor fully soft, just in between. It connects the hardware with another software that needs the hardware's function.

Yin and Yang

Hardware is a philharmonic concert hall full of musical instruments. Software is a conductor and musical instrument players. Computer needs both to play a music. Hardwares enable softwares, softwares utilize hardwares. Without hardwares, your softwares can't take action because there is no physical setting for that. Without softwares, your hardwares can't function because there is no operator for that.

Source matters

Now we can finally talk about what the word "open" means in computers and computing.

Remember what source code is? Source code is a set of instruction written in a human-readable programming language. Source code is the most original and essential element to make a software. It's a blueprint of how the software is supposed to operate.

Just like creative works such as artwork, literature, and choreography, source code itself and the software built from it are intellectual creation so it is protected under the copyright law. You cannot see, use, share, and change the source code and the software without the creator's legal permission. The term "proprietary software" is used to describe such private, restricted, and protected software and its source code. Even though the source code is available, either partially or wholly, either to the public or few select people, it is still considered proprietary if there is a restriction in public use, modification, and distribution.

Open to the public

Open source means the creator of the software — who is also a copyright holder — allows anyone to see and share the source code for any purpose. There are different sets of permissions that define what is allowed to do with the source code, and we call it "license". For example, an open source license called MIT License looks like this.

Copyright (c) [year] [copyright holders]

Permission is hereby granted, free of charge, to any person obtaining a copy of this software and associated documentation files (the "Software"), to deal in the Software without restriction, including without limitation the rights to use, copy, modify, merge, publish, distribute, sublicense, and/or sell copies of the Software, and to permit persons to whom the Software is furnished to do so, subject to the following conditions:

The above copyright notice and this permission notice shall be included in all copies or substantial portions of the Software.

THE SOFTWARE IS PROVIDED "AS IS", WITHOUT WARRANTY OF ANY KIND, EXPRESS OR IMPLIED, INCLUDING BUT NOT LIMITED TO THE WARRANTIES OF MERCHANTABILITY, FITNESS FOR A PARTICULAR PURPOSE AND NONINFRINGEMENT. IN NO EVENT SHALL THE AUTHORS OR COPYRIGHT HOLDERS BE LIABLE FOR ANY CLAIM, DAMAGES OR OTHER LIABILITY, WHETHER IN AN ACTION OF CONTRACT, TORT OR OTHERWISE, ARISING FROM, OUT OF OR IN CONNECTION WITH THE SOFTWARE OR THE USE OR OTHER DEALINGS IN THE SOFTWARE.

The key point of the concept "open source" is public availability and accessibility of the source code. Open source licenses guarantees the access to the source code for anyone to freely see, audit, share, and study it. But when it comes to its use, modification, redistribution, and derivative works, certain open source licenses might restrict some of the freedom you can do with it.

Open, is not equal to freedom.

Free as in freedom

"Free software" is a term for open source softwares which its creator ensures everyone to audit, study, use, modify, redistribute, improve the software and its source code for any purpose, via "free software license". Generally, free software licenses ensure wider range of freedom compared to open source licenses, except when the license enforces your derivative work to have the same license. This clause is known as "copyleft" which is an opposing concept to "copyright". Copyleft license is invented to encourage collaborative prosperity of free softwares, but some people might find this restrictive. In this sense, the "freedom" here is not indulgence but philosophical orientation.

See also: Free software and its philosophy

Note that the word "free" here doesn't mean "free of charge". Most free softwares are available for free because of its philanthropistic nature, but there are commercial for-profit free softwares as well. Keep in mind, this "free" means "freedom". This is probably the biggest difference between open source software and free software. Free software strongly emphasizes on freedom.

Hardware can be open too

Previously we talked about what hardware is. How can hardware be open? As you already know, hardware is made of physical parts and has a tangible form. So open source hardware means not only the software that runs such hardware is open to public, but also hardware's electronical and mechanical structure is. Usually detailed circuitry, schematic, and assembly is documented and published along with the source code. So if you have components and equipments ready, you can build the hardware by yourself. Due to the cost, complexity and difficulty of hardware making process, open source hardwares are less common and accessible than open source softwares. But Opencomputings is here to provide easy-to-digest information about entry-level open source hardwares that we all can use.
